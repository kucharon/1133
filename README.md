# 回收管理系统 - 业务员/门店端

## 项目简介

这是一个电子电器回收管理系统的业务员/门店端演示界面，使用HTML、CSS和JavaScript实现，模拟微信小程序的界面风格。系统主要功能包括订单管理、客户管理、新增订单等操作，旨在为回收行业的业务员和门店提供便捷的管理工具。

## 功能特点

### 1. 完整的页面体系
- **登录页面**：账号密码登录
- **首页**：统计卡片、快捷操作、最近订单
- **订单管理**：订单列表、状态筛选
- **订单详情**：订单信息、客户信息、回收信息
- **新增订单**：客户信息录入、回收信息填写、订单来源选择
- **客户管理**：客户列表、搜索功能
- **个人中心**：用户信息、系统设置

### 2. 核心功能
- **订单管理**：查看、筛选、创建订单
- **客户管理**：查看客户信息
- **订单来源区分**：支持业务员和门店两种来源
- **业务员标注**：订单来源为业务员时，显示业务员姓名
- **表单验证**：新增订单时的表单验证
- **自动计算**：根据重量和单价自动计算回收金额

### 3. 界面特点
- **模拟微信小程序风格**：底部导航栏、卡片式布局
- **响应式设计**：适配移动设备屏幕
- **美观的视觉效果**：现代化的UI设计
- **流畅的交互体验**：加载动画、提示框、确认对话框

## 如何使用

1. **直接打开**：在浏览器中直接打开 `login.html` 文件
2. **登录**：输入任意账号和密码，点击登录按钮
3. **浏览功能**：通过底部导航栏切换不同页面
4. **新增订单**：点击首页的"新增订单"按钮或订单页面的"+"按钮
5. **查看订单**：在订单页面点击订单卡片查看详情
6. **管理客户**：在客户页面查看客户列表

## 演示说明

### 登录演示
- 账号：任意输入
- 密码：任意输入
- 点击登录按钮即可进入系统

### 订单来源功能
- **业务员来源**：在新增订单时选择"业务员"，会显示业务员姓名输入框
- **门店来源**：在新增订单时选择"门店"，业务员姓名输入框会自动隐藏
- **订单列表**：业务员来源的订单会显示业务员姓名
- **订单详情**：会显示订单来源和业务员信息

### 新增订单流程
1. 填写客户信息（姓名、电话、地址）
2. 选择回收品类（会自动填充默认单价）
3. 输入重量
4. 系统自动计算金额
5. 选择订单来源
6. （如果选择业务员）输入业务员姓名
7. 点击提交订单

## 注意事项

- 本项目为演示版本，所有数据均为模拟数据
- 部分功能（如扫码回收、打印订单等）仅做了界面展示，实际功能未实现
- 项目使用纯前端技术实现，没有后端服务
- 建议在现代浏览器中打开，如Chrome、Firefox、Safari等

## 部署说明

1. 将 `miniprogram-demo` 目录上传到GitHub仓库
2. 在GitHub仓库的Settings中启用GitHub Pages
3. 选择 `miniprogram-demo` 目录作为源
4. 等待部署完成后，即可通过生成的URL访问演示页面
